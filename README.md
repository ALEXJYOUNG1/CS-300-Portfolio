## Reflection

### 1. What was the problem you were solving in the projects for this course?
In this course, I solved the problem of organizing and retrieving course data efficiently 
for an advising-style application. The projects required me to compare data structures and then implement 
a working C++ program that could load course information, sort it in alphanumeric order, and display 
course details with prerequisites. The core challenge was building a solution that was accurate, efficient, and easy to follow.

### 2. How did you approach the problem? Consider why data structures are important to understand.
I approached the problem by first analyzing different data structures based on run-time and memory usage, 
then using that analysis to guide my implementation. Data structures are important because they affect how quickly a program can search, store, and organize information. This course helped me think beyond just writing code and focus more on making good design choices for performance and scalability.

### 3. How did you overcome any roadblocks you encountered while going through the activities or project?
I overcame roadblocks by breaking the program into smaller parts and testing each part separately. 
I worked through file input, parsing, sorting, searching, and output formatting step by step. When something failed, 
I used repeated testing and debugging to isolate the issue before changing more code. That approach made the 
project easier to manage and helped me avoid introducing new errors while fixing old ones.

### 4. How has your work on this project expanded your approach to designing software and developing programs?
This project expanded my approach by making me think more carefully about software design before writing code. 
I now pay more attention to how data flows through a program, what structure is best for the problem, 
and how the user will interact with the application. It reinforced that software development is not only about
making something work, but also about building it in a way that is efficient and organized.

### 5. How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
My work in this course improved how I write maintainable code by reinforcing modular design, readable function organization,
and consistent formatting. I learned the value of separating tasks into clear functions and using names that make the code
easier to understand later. It also improved my focus on adaptability, because well-structured code is easier to update when
requirements change or new features are added.
